# YJackGameStudio vs. Raw AI Coding Tools

## Purpose

This page helps readers decide whether they need YJackGameStudio or whether
using an AI coding tool directly is a better fit.

This is a scope comparison, not a claim that raw AI coding tools are inferior.
Many teams should start with a direct tool workflow and only add structure when
they actually need it.

## What "Raw AI Coding Tools" Means Here

This page refers to using tools such as Codex, GitHub Copilot, Gemini, Google
Antigravity, or Claude Code directly without a shared studio operating model on
top of them.

That direct approach can be completely valid. The comparison is about workflow
shape, review discipline, and portability, not about whether the underlying
tools are useful.

## Use YJackGameStudio When...

- You want more than one assistant persona and need explicit role separation
  across design, implementation, QA, production, and release work.
- You want reusable procedural workflows instead of reinventing process in each
  session.
- You want owner-directed autonomy with written hard gates and escalation
  boundaries.
- You want validation language that distinguishes between checks that ran,
  checks that did not run, and manual validation still required.
- You want a repository other contributors can inspect, fork, and adapt as a public
  operating model.
- You want game-development-specific structure rather than a general-purpose
  coding assistant workflow.

## Use Another Tool When...

- You mainly want file edits, debugging help, or ad hoc coding assistance
  without adopting a larger workflow system.
- Your team already has its own production process and does not need a public
  reference architecture layered on top.
- You want the lowest setup overhead and prefer direct prompting over structured
  roles, skills, and review gates.
- You are not building game-development workflows and do not need engine-aware
  or studio-oriented guidance.

## Key Difference in Positioning

Raw AI coding tools are execution surfaces. YJackGameStudio is the workflow
layer that sits around those tools and defines roles, contracts, approvals,
validation expectations, and reusable game-development processes.

YJackGameStudio does not replace those tools. It organizes how they are used in
an owner-directed game studio workflow.

## What This Comparison Does Not Claim

- It does not claim raw AI coding tools cannot support disciplined workflows.
- It does not claim YJackGameStudio improves model quality.
- It does not claim one tool provider is required.
- It does not claim hosted orchestration, autonomous game generation, or asset
  generation exists in this repository.
- It does not claim this repository is a hosted product rather than a public
  reference architecture.
- It does not claim Loomlight Studio behavior exists in this repository.
- It does not claim YJackCore is required.
